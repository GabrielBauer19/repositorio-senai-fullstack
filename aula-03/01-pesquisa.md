1. Estrutura condicional
Programa tomando decisão. Se chover, pego guarda-chuva. Senão, não pego.

2. if vs if/else
Só if: se for falso, não acontece nada. Com else: sempre tem uma resposta.

3. else if
Mais de dois caminhos. Nota 9+ ótimo, 7+ bom, 5+ regular, senão ruim.

4. true ou false
Resposta de sim ou não. "Restaurante aberto?" — só pode ser um dos dois.

5. **== vs ===**  O == aceita formatos diferentes com mesmo valor. === exige exatamente o que foi pedido. Use sempre.

6. &&, ||, !
&& precisa dos dois: cartão e senha. || basta um: ingresso ou nome na lista. ! inverte: se não pagou, bloqueia.

7. Operador %
Resto da divisão. 14 ovos em caixas de 6, sobram 2. Se dividir por 2 e não sobrar nada, é par.

8. switch vs else if
switch pra opções fixas e conhecidas. else if pra faixas de valores e condições complexas.

9. break e default
break encerra o caso atual. Sem ele, executa os próximos sem querer. default responde quando nenhuma opção bateu.

10. Por que testar com vários valores?
Elevador testado com uma pessoa, aprovado. Entraram seis, travou. Teste o caso normal, o limite e as entradas erradas.
